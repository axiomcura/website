Required stack

To use Jekyll, you need Ruby, Gems, and Jekyll. This can be easily installed in Homebrew.

Use this website on how to install Ruby and Jekyll:

  • https://jekyllrb.com/docs/installation/macos/

Trouble shooting

  • Getting developments error:
    echo "source $(brew --prefix)/opt/chruby/share/chruby/chruby.sh" >> ~/.zshrc
echo "source $(brew --prefix)/opt/chruby/share/chruby/auto.sh" >> ~/.zshrc
echo "chruby ruby-3.1.2" >> ~/.zshrc
    • Make sure that the current version of ruby under chruby is correct. To check just type chruby and it will return a version number. use that as the input for ruby under the chruby call
  • When trying to start the server and it fails try to do this (source):
bundle add webrick

Creating a site with Jekyll

Create a GitHub repository with a README.md file. Then go to the settings and click on pages and set up your GitHub pages. Go back to the pages settings and copy the link to your webpage.

You can edit the ReadME file to generate the website.

by typing:

jekyll new . --force

This will create a number of different files, which will be working with

To see the website type:

bundle exec jekyll serve --livereload
  • The --livereload indicates that any changes applied will reload automatically.

Follow these instructions for gh-pages

# If you want to use GitHub Pages, remove the "gem "jekyll"" above and
# uncomment the line below. To upgrade, run `bundle update github-pages`.

Then type bundle update

If there is an error occuring, remove the Gemfile.lock as it is causing the error

Now the next steps is to get it running on github

  • First you need to update the baseurl and url in the _config.yaml file
    • You will need to use your website as your url and the name of the website (ending portion) and the baseurl
    baseurl: "/website-test" # the subpath of your site, e.g. /blog
url: "https://axiomcura.github.io" # the base hostname & protocol for your site, e.g. http://example.com
